# Security Policy

## Supported Versions

Security fixes are accepted for the latest `0.1.x` release line.

## Reporting a Vulnerability

Please report suspected vulnerabilities through GitHub Security Advisories or by opening a minimal public issue without private data, credentials, or exploit details.

Include the affected version, the command or fixture shape involved, expected impact, and any safe reproduction steps. Maintainers will confirm receipt, triage the report, and publish remediation notes when a fix is available.

## Scope

`tool-demo-script-skill` is a local fixture-to-script generator. Reports are in scope when they involve unsafe file handling, unintended command execution, package integrity, or misleading safety boundaries in generated demo plans.
